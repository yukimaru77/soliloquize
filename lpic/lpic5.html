パッケージ

パッケージとは

パッケージとは実行プログラムや設定ファイル、ドキュメントやマニュアルなどの関連するファイルを一つにまとめたものを言います

パッケージの種類

パッケージにも色々な種類があってDebian GNU/Linuxなどで採用されているdeb形式、Red Hat Enterprise Linuxなどで採用されているrpm形式(Red Hat Package Manager)などがある。

パッケージの扱い方

パッケージは基本的に違う形式同士では互換性がありません。なのでそれぞれのパッケージは専用のコマンドを使って扱います。deb形式はdpkgコマンド(Debian package manager)及びdpkgコマンドのフロントエンドとして機能するAPTコマンド(apt-getコマンド及びapt-cacheコマンド)、rpm形式ではrpmコマンド及びyumコマンド(Yellow dog Updater Modified)があります

dpkgコマンド

dpkgコマンドはあくまでdeb形式のファイルを操作するコマンドなのでdebファイルは前もってネットなどからダウンロードする必要があります。

なおdebファイルの名前には規則がありunzip_6.0-16+deb8u2_amd64.debなら

unzip

パッケージ名

6.0

バージョン番号

16+deb8u2

リビジョン番号

amd64

アーキテクチャ

dpkgコマンドの書式は以下になります

dpkg [オプション] アクション
オプション/アクション 説明
-i(--install) パッケージファイル名 パッケージをインストールする
-r(--remove) パッケージ名 設定ファイル以外をアンインストールする
-p(--purge) パッケージ名 設定ファイルも含めてすべてアンインストールする
-l(--list) パッケージ名検索パターン インストール済みのパッケージの中から検索する
-S(--search) ファイル名検索パターン 指定したファイルがどのパッケージからインストールされたかを表示する
-L(--listfiles) パッケージ名 指定したパッケージからインストールされたファイル一覧を表示する
-s(--status) パッケージ名 パッケージの情報を表示する
-E 同じバージョンのパッケージがあるならインストールしない
-G 新バージョンがインストールされているならインストールしない

このコマンドのめんどくさい点はパッケージに依存関係があるとそれをインストールするまでエラーになる、(バージョンが上がるたびに)パッケージファイルは自分で探してダウンロードしてこなければいけない、という点です。

そこでAPT(Advaced Packaging Tool)というパッケージ管理ツールに含まれるコマンドを使用します。

apt-getコマンド

apt-get [オプション] サブコマンド
オプション/アクション 説明
update パッケージデータベースを/etc/apt/sources.listに記載されたURLから更新する
upgrade システムの全パッケージのうち、他のパッケージを削除しないものをアップグレードする
dist-upgrade システムを最新にアップグレードする
install パッケージ名 パッケージをインストール、アップグレードする
remove パッケージをアンインストールする
clean 過去に取得して保持したパッケージファイルを削除する。
-d ファイルのダウンロードのみを行う
