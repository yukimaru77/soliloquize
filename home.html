Linuxの起動

電源を入れた後の流れ

電源を入れるとパソコンには画面が表示されるわけですが実はいろいろな工程を経てパソコンに画面が表示されています。

具体的には 電源を入れる → BIOS/UEFI実行 → ブートローダー → カーネルの読み込み → SysVinit/systemd(プロセス起動) → ログイン可能 の順で起動します。

よくわからない単語がいっぱい出てきたと思うので順を追って説明していきます。

BIOSとは？

BIOS(Basic Input Output System:入出力基本システム)は、パソコンに接続されたハードウェアを制御するシステムで、ハードウェアとソフトウェアの仲介役をしています。システムを初期化し使用可能な状態にクリーンアップしてくれてます。機種によっては進化バージョンのUEFIを使ってる場合もあります。

BIOSのセットアップをするには機種によってさまざまだですがDeleteキーやF1キー、F2キーなどを押すとセットアップ画面が表示されます。

  • 日付や時刻の変更
  • 起動順位の変更
  • 各種デバイスの使用・不使用
    • などがある。(他にもいろいろ機能はあります)

余談:起動順位

起動順位とは文字通りどのハードウェアを起動するかの順番です。デフォルトではSSD・HDDか光学ディスクになって、さらにそのハードウェアにOS起動ファイルがあれば実行されます。(少し違うので後で補正を入れますが)

例えばSSD・HDDのOS起動ファイルがwindowsとなっています。なので修復ディスクなどでシステムを修復する場合は修復するを入れたハードウェアの起動順位をSSDより先にしなければないらない。

ブートローダーとは?

ブートローダーとはパーティションテーブルを読んで、ブートセレクタを参照するプログラムです。。。分かりにくい…; ;

ブートセレクタは各パーティションの読み込まれる先頭の場所でありOSの起動ファイルなどが入っています。それによってカーネルがメモリに読み込まれます。ですがここまでの流れを世間一般的にはブートローダーといいます。ブート(起動)をメモリにロード(読み込む)のでブートローダーです。要はカーネルをメモリに読み込むやつです。

より詳細(厳密)な説明

順番的には以下のように実行されます

BIOS → ハードウェアのマスターブートレコードへ → その中のブートローダー起動 → パーティションテーブル(を参照) → パーティション内のブートセレクタ → (その中にOS起動ファイルがあれば)OS起動

マスターブートレコードとは？
ハードディスクの先頭セクタでブートローダーやパーティションテーブルが格納されている場所のことを言います。
ブートセレクタとは？
パーティションの先頭セクタでOS起動ファイルなどが格納されている場所のことを言います。

SysVinit

SysVinit(System Five Init)はLinuxのOS起動の仕組の名前です。(5番目に位置するからSysVinit…？)

具体的な仕組みは長くなるので別ページで詳しくやります。

とりあえずカーネルがメモリに読み込まれた後SysVinitによる起動が行われログインできるようになると思っておけばいいです。
